# AlphaScanner 极简选股预警器 — 使用手册

---

## 项目简介

**AlphaScanner** 是一款基于 Python + Streamlit 构建的 A 股多因子选股与技术信号预警工具。系统自动扫描全市场约 5800 只股票，通过动量、估值、波动、ROE 四维因子模型打分排名，筛选出 Top 20 候选股票，并结合均线系统与布林带技术指标，实时生成建仓 / 风险预警信号。

**核心特点：**
- **零依赖第三方数据服务**：直接对接东方财富、腾讯财经公开 API，无需注册任何账号
- **全自动扫描**：一键获取全市场行情 + 财务数据，自动缓存当日结果
- **5 分钟自动刷新**：交易时段内自动更新行情数据，无需手动刷新
- **四因子选股模型**：动量 30% + 估值 30% + 波动 20% + ROE 20% 综合打分
- **技术指标可视化**：MA5/MA20 均线 + 布林带 + 金叉死叉信号一图呈现
- **纯前端展示**：Streamlit 单页应用，本地运行，开箱即用

**技术栈：** Python 3.9+ / Streamlit / pandas / mplfinance / urllib

## 三、操作流程

### 第 1 步：加载数据

点击左侧 **「🔄 刷新全市场数据」** 按钮，系统将依次执行：

1. 获取全市场股票列表（约 5000 只）
2. 获取 ROE 财务数据
3. 计算多因子得分并排名

> ⏱️ 首次运行约需 1~3 分钟，请耐心等待。  
> 💾 扫描结果会自动缓存，**当天再次打开无需重新扫描**。

### 第 2 步：查看候选池

扫描完成后，左侧表格展示 Top 20 股票：

| 字段       | 含义                               |
| ---------- | ---------------------------------- |
| 代码       | 股票代码（如 000001）              |
| 名称       | 股票简称                           |
| 得分       | 四因子加权综合得分（越高越好）     |
| 20日涨幅%  | 近 20 个交易日累计涨跌幅           |
| PB         | 市净率（越低越好）                 |
| ROE        | 净资产收益率（%，>8% 额外加分）    |

### 第 3 步：查看 K 线图

**点击表格中任意一行**，右侧会自动切换显示该股票的 K 线图。

K 线图上包含：
- 🟠 **橙线**：MA5 短期均线
- 🔵 **蓝线**：MA20 长期均线
- ⬜ **灰色虚线**：布林带上下轨（±2倍标准差）
- 🟣 **紫色点线**：布林中轨

### 第 4 步：判断信号

K 线图上方会显示当前信号状态：

| 信号颜色 | 含义         | 操作建议             |
| -------- | ------------ | -------------------- |
| 🟢 绿色  | **建仓信号** | MA5上穿MA20（金叉）或突破布林上轨 |
| 🔴 红色  | **风险信号** | MA5下穿MA20（死叉）或跌破布林下轨 |
| ⚪ 白色  | **观望**     | 无明显信号，继续等待  |

K 线图上也会用箭头标注：
- **绿色▲**：买入点（B）
- **红色▼**：卖出点（S）

## 四、数据来源与选股逻辑

### 数据来源

| 数据类型     | 数据源                           | 说明                         |
| ------------ | -------------------------------- | ---------------------------- |
| 股票列表     | 东方财富 datacenter API          | 全 A 股代码与名称            |
| 实时行情快照 | 腾讯财经 API (qt.gtimg.cn)       | 收盘价、PB、20日涨幅、换手率等 |
| ROE 财务数据 | 东方财富业绩报表 API             | 加权平均净资产收益率         |
| 日线 K 线    | 腾讯财经 API (web.ifzq.gtimg.cn) | 前复权 OHLCV 日线数据        |

> 所有数据均来自公开 API，无需注册账号，交易时段获取实时数据，非交易时段使用缓存。

### 选股逻辑

系统使用 **四因子模型** 对全市场股票打分排名：

| 因子       | 权重 | 说明                               |
| ---------- | ---- | ---------------------------------- |
| 动量因子   | 30%  | 20日涨幅，涨幅越大得分越高         |
| 估值因子   | 30%  | 市净率PB，PB越低得分越高           |
| 波动因子   | 20%  | ATR活跃度，剔除成交低迷的僵尸股    |
| ROE因子    | 20%  | 净资产收益率，>8% 额外加分         |

综合得分 = 各因子百分排名 × 权重，取 Top 20 进入候选池。

### 信号检测逻辑

基于 MA5/MA20 均线系统与布林带（±2σ）生成交易信号：

| 信号类型   | 触发条件                                   |
| ---------- | ------------------------------------------ |
| 突破建仓   | MA5 上穿 MA20（金叉）**且** 收盘价突破布林上轨 |
| 金叉建仓   | MA5 上穿 MA20（金叉）                      |
| 死叉离场   | MA5 下穿 MA20（死叉）                      |
| 跌破预警   | 收盘价跌破布林带下轨                       |

## 五、常见问题

### Q1：点击刷新后一直转圈？
A：首次扫描需要获取约 5000 只股票的日线数据，大约需要 1~3 分钟，请耐心等待。

### Q2：提示"数据不足，无法绘制K线图"？
A：该股票上市时间较短或停牌，数据不足 30 天，请换一只股票查看。

### Q3：如何更新到最新数据？
A：点击「刷新全市场数据」按钮即可强制重新扫描。如果当天已扫描过，结果会自动缓存。

### Q4：ROE 数据获取失败？
A：ROE 数据来源于东方财富业绩报表接口，非交易时间或数据更新期间可能暂时为空，不影响其他功能正常使用。

### Q5：报错 ConnectionError？
A：请检查网络连接是否正常，确保可以访问东方财富网站。

## 七、参数调整（进阶）

如需调整选股参数，编辑 `config.py`：

```python
MA_SHORT = 5        # 短期均线天数
MA_LONG = 20        # 长期均线天数
BOLL_PERIOD = 20    # 布林带周期
BOLL_STD = 2        # 布林带标准差倍数
TOP_N = 10          # 候选池股票数量
ROE_THRESHOLD = 8.0 # ROE 加分阈值（%）

FACTOR_WEIGHTS = {
    "momentum": 0.3,    # 动量因子权重
    "valuation": 0.3,   # 估值因子权重
    "volatility": 0.2,  # 波动因子权重
    "roe": 0.2,         # ROE因子权重
}
```

修改后需重启应用生效。

## 八、缓存机制

系统采用本地 CSV 文件缓存策略，减少重复请求：

| 缓存文件          | 内容             | 失效规则                           |
| ----------------- | ---------------- | ---------------------------------- |
| `stock_list.csv`  | 全市场行情快照   | 当日有效；交易时段内不使用缓存     |
| `roe_data.csv`    | ROE 财务数据     | 当日有效                           |
| `daily_XXXXXX.csv`| 单只股票日线数据 | 当日有效；交易时段内不使用缓存     |
| `today_pool.csv`  | 候选池结果       | 当日有效；5 分钟自动刷新后重新生成 |

> 点击「刷新全市场数据」按钮会强制清除缓存并重新扫描。
